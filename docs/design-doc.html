设计意图

Agent Core 是一个薄内核：负责 run、event、state、registry、approval 这些稳定原语。飞书、workflow、多 agent、dashboard、重沙盒、eval 都是外部能力。

目标不是做一个大平台，而是给 Agent 自己写外部程序和服务留下可靠的挂载点。

约束

  • 单个手写文件不超过 500 行。
  • 单目录不超过 20 个文件。
  • Node.js workspace 深度不超过 6 层。
  • 不要提交 secret、`.env`、本地 state 或原始敏感日志。

评审结论

继续保持薄实现。 M4b 已完成 chat-only 飞书模型回复；下一阶段补持久 inbox/outbox 和长连接进程。

长期协议记录在 Architecture RFC；当前实现只做真实阶段已经证明必要的机制。

未执行独立 subagent 审查：当前工具规则要求只有用户明确要求委派时才能调用。
